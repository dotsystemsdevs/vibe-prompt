# From Generic Ui To Brand Distinct Interface

## Purpose
Upgrade a functional but generic AI-generated UI into a distinct, brand-consistent product interface.

## Input
- Current UI screenshots/components
- Brand direction (tone, references, positioning)
- Product context and core user journeys

## Instructions
1. Diagnose why the current UI feels generic (layout, hierarchy, spacing, visual language).
2. Define a compact design identity system (typography, color, motion, surface rules).
3. Prioritize high-impact UI upgrades by user journey.
4. Produce component-level upgrade instructions for hero areas and key workflows.
5. Add anti-template rules to avoid default "shadcn-like" sameness.
6. Output a phased redesign plan with measurable quality checkpoints.

## Output Format
- `Generic UI Diagnosis`
- `Design Identity Rules`
- `Priority Upgrade Backlog`
- `Component Upgrade Specs`
- `Anti-Template Guardrails`
- `Phased Redesign Plan`

## Quality Criteria
- Distinct but usable interface
- Strong hierarchy and readability
- Consistent visual system
- Practical for iterative implementation

## Variants
- Variant A: Fast polish for MVP launch
- Variant B: Full brand-level UI overhaul

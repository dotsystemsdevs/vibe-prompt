# Frontend Form Input Prompt Pack

## Purpose
Create robust form/input components with good UX, validation and accessibility.

## Input
- Form type
- Fields and validation rules
- Unwanted interaction

## Instructions
Copy the prompt below:

```text
Generate a form/input component pack.

Form context:
[insert context]

Stack:
React + Tailwind + React Hook Form + Zod

Include:
1) Multi-step form pattern
2) Real-time validation behavior
3) Success/error/loading states
4) Keyboard accessibility
5) Reusable input primitives (text, select, date, upload, tags)

Requirements:
- Validate on blur and submit
- Clear field-level messages
- Prevent layout shift on errors
- Mobile-friendly controls

Output format:
[Form Architecture]
...

[Validation Rules]
...

[UI States]
- default:
- error:
- success:
- loading:

[Reusable Primitives]
- ...
```

## Output Format
- Formarkitektur
- Valideringsspec
- UI-state rules + inputprimitives

## Quality Criteria
- Less broken form UX
- Higher completion rate
- Easy to recycle in several rivers

## Variants
- Variant A: Signup/onboarding.
- Variant B: Checkout/payment.

# Frontend Four Part Shipping Framework

## Purpose
Create frontend prompts that actually deliver through a clear 4-part structure.

## Input
- Identity (what is built)
- Audience (who it is for)
- Features (functionality)
- Aesthetic constraints (visual style + technique)

## Instructions
Copy the prompt below:

```text
Build this UI using the 4-part framework.

Identity:
[what to build]

Audience:
[who this is for]

Features:
- [feature 1]
- [feature 2]
- [feature 3]

Aesthetic + technical constraints:
- Design style:
- Colors:
- Typography:
- Framework/libraries:
- Responsive behavior:

Requirements:
1) Generate production-lean component structure.
2) Include accessibility basics.
3) Keep style consistent with constraints.
4) Add quick verify checklist.

Output format:
[UI Spec]
- ...

[Implementation]
...

[Verify Checklist]
- ...
```

## Output Format
- UI-spec
- Implementation
- Verifieringschecklista

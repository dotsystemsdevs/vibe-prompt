# Solo Builder Routine Build Break Fix Improve

## Purpose
Operationalize the real solo vibe-coding cycle: ship quickly, discover mess, recover quality, then compound improvements.

## Input
- Current app status
- Available weekly build hours
- Quality targets (reliability, performance, maintainability)

## Instructions
1. Split work into four loops: build, break discovery, fix/harden, improve.
2. Define expected time budget per loop phase for realistic planning.
3. Add quality gates before and after first real user testing.
4. Include a "mess recovery" protocol for post-launch surprises.
5. Build a weekly compounding improvement backlog.
6. Output a repeatable operating routine and stop criteria.

## Output Format
- `4-Loop Operating Model`
- `Time Budget Allocation`
- `Quality Gate Checklist`
- `Mess Recovery Protocol`
- `Compounding Improvement Backlog`
- `Stop/Continue Rules`

## Quality Criteria
- Reflects real solo-builder constraints
- Prevents burnout from chaotic iteration
- Keeps momentum while improving quality
- Reusable week after week

## Variants
- Variant A: Part-time indie builder
- Variant B: Full-time solo founder

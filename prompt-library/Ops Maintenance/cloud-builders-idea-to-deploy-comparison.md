# Cloud Builders: Idea-to-Deploy Comparison

## Purpose

Create a prompt that helps you choose the right cloud-based builder to go from idea to deployment with minimal friction.

## Input

- Product idea
- Technical requirements (frontend, full stack, mobile)
- Time-to-demo and budget

## Instructions

1. Classify the need: rapid prototype, internal app, or customer-ready product.
2. Compare cloud builders on speed, control, exportability, and deployment flow.
3. Recommend a primary choice and a backup choice.
4. Describe a step-by-step plan for first delivery within 24–72 hours.
5. List lock-in risks and how to mitigate them.
6. Finish with a copy-ready starter prompt for the selected tool.

## Output Format

- `Project Profile`
- `Tool Comparison Matrix`
- `Recommended Path`
- `72h Delivery Plan`
- `Lock-in Mitigation`
- `Starter Prompt`

## Quality Criteria

- Decision grounded in real constraints
- Focus on shipping, not feature lists alone
- Clear fallback if the primary tool fails
- Immediately usable in practice

## Variants

- **Variant A:** Fast MVP with maximum speed.
- **Variant B:** Foundation-first with stronger long-term control.

# Prompt Library Versioning

## Purpose

Avoid prompt chaos by saving, versioning, and tracking test status for prompts as if they were code.

## Input

- List of existing prompts
- Use cases
- Test results and notes

## Instructions

Copy the prompt below:

```text
Help me organize my prompts like code.

Tasks:
1) Create a folder taxonomy by category and use case.
2) Propose a file naming convention.
3) Define version tags (v1, v1.1, v2) and changelog style.
4) Add metadata template:
   - purpose
   - input shape
   - output format
   - tested on
   - pass/fail notes
5) Suggest a weekly maintenance routine.

Output format:
[Folder Structure]
...

[Naming Convention]
...

[Metadata Template]
...

[Weekly Maintenance]
...
```

## Output Format

- Structure for the prompt library
- Versioning rules and metadata
- Weekly maintenance routine

## Quality Criteria

- Easy to find the right prompt
- Traceable evolution of prompts
- Less duplication and forgotten variants

## Variants

- **Variant A:** Solo workflow in Markdown.
- **Variant B:** Team workflow with Git and review.

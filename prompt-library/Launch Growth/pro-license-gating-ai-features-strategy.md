# Pro License Gating Ai Features Strategy

## Purpose
Design a free vs pro AI feature-gating strategy that protects API costs, preserves user trust, and improves conversion without harming product experience.

## Input
Use this prompt when you need to package AI features into clear free and paid tiers with predictable cost and upgrade paths.

- Feature inventory with business value and usage frequency
- Current or target free vs pro positioning
- Desired conversion model (trial, soft paywall, hard gate, usage caps)
- Technical constraints (model limits, API cost ceiling, latency, reliability)
- Product context (audience, use case, retention goals)

## Instructions
1. Divide functions into free baseline, pro core and demo level.
2. Set clear boundaries for what is free vs paid.
3. Define frictionless onboarding to pro without dark patterns.
4. Describe how demo keys or trial should provide a taster with control.
5. Include pricing logic at concept level and risks in case of wrong gating strategy.
6. End with KPI proposals for retention and conversion.

## Output Format
- `Feature Tier Matrix`
- `Activation Flow`
- `Trial/Demo Logic`
- `Risk & Ethics Notes`
- `KPI Proposal`

## Quality Criteria
- Well-scoped division between free and pro
- Clear communication to the user
- Feasible API cost profile
- Measurable impact on conversion

## Variants
- Variant A: Consumer/community product (high free value, soft upsell)
- Variant B: B2B/internal product (stronger gate, clear premium ROI)

# Vibe To Customer Shipping Feedback Loop

## Purpose
Convert vibe-coding output into a customer-facing feedback loop instead of endless local iteration.

## Input
- Feature or prototype state
- Target customer profile
- Current unknowns (value, usability, pricing, retention)

## Instructions
1. Define the smallest customer-visible slice that can ship this week.
2. Build a feedback plan with 3-5 customer questions tied to business risk.
3. Create a lightweight release checklist (quality, telemetry, rollback, support note).
4. Specify what evidence counts as "validated" vs "invalidated."
5. Convert feedback outcomes into next sprint prompt tasks.
6. Prevent overbuilding by enforcing a "no signal, no scale" rule.

## Output Format
- `Minimum Shippable Slice`
- `Customer Feedback Questions`
- `Release Checklist`
- `Validation Criteria`
- `Next Sprint Prompt Backlog`
- `No-Signal Guardrail`

## Quality Criteria
- Customer learning prioritized over code volume
- Fast path from build to evidence
- Actionable feedback instrumentation
- Clear anti-overengineering guardrails

## Variants
- Variant A: B2C rapid feedback
- Variant B: B2B stakeholder interview flow

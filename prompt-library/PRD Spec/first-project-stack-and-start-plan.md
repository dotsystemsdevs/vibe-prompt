# First Project Stack And Start Plan

## Purpose
Help non-coders choose the right stack and first implementation path for a real project without getting overwhelmed.

## Input
- Project idea
- Current technical background
- Time available per week

## Instructions
1. Translate the project idea into core feature blocks.
2. Recommend a beginner-safe stack with rationale.
3. Define what to learn first vs what to delegate to AI.
4. Create a milestone plan from prototype to usable v1.
5. Add learning checkpoints tied to shipped functionality.
6. Include escalation triggers for when to involve a senior developer.

## Output Format
- `Feature Block Map`
- `Recommended Stack`
- `Learn vs Delegate Matrix`
- `Milestone Plan`
- `Learning Checkpoints`
- `Senior Dev Escalation Triggers`

## Quality Criteria
- Beginner-friendly without being naive
- Keeps shipping and learning aligned
- Reduces stack confusion and rework
- Encourages strong fundamentals over blind prompting

## Variants
- Variant A: Web app first project
- Variant B: Mobile-first project

# Build Decision Pivot Persist Stop

## Purpose
Provide a decision framework prompt for "Should I keep going, pivot, or stop?" when vibe-coding feels broken.

## Input
- Current project status
- Customer evidence (or lack of evidence)
- Burnout level, time runway, and confidence level

## Instructions
1. Assess project state across four axes: user value, technical viability, energy sustainability, and market signal.
2. Identify whether the core issue is product, execution, skill gap, or expectations.
3. Score options: persist, pivot, pause, or stop.
4. If "persist" or "pivot," define a 14-day action plan with measurable checkpoints.
5. If "pause" or "stop," define knowledge capture and next-step recovery.
6. Return a single recommendation with rationale and risk.

## Output Format
- `Project State Snapshot`
- `Root Cause Diagnosis`
- `Option Scorecard`
- `Recommended Path`
- `14-Day Plan or Exit Plan`
- `Risk/Assumption Log`

## Quality Criteria
- Honest, unsentimental assessment
- Decision tied to evidence
- Clear next action in every outcome
- Reduces indecision loops

## Variants
- Variant A: Solo founder mode
- Variant B: Team alignment mode

# Prompt Draft Workflow

Du skickar artiklar -> jag tar ut det viktiga -> jag skriver/uppdaterar prompts.

## Filnamn

Alla prompt-filer ska heta:

`kategori-promtnamn.md`

Exempel:

- `marknadsforing-viral-hook-generator.md`
- `produkt-copy-landingpage-v1.md`
- `content-youtube-outline-multi-angle.md`

## Hur jag jobbar pa varje artikel

1. Identifierar huvudide, vinkel, tonalitet och malgrupp.
2. Extraherar anvandbara "byggblock" (ramverk, argument, CTA, motargument, struktur).
3. Skapar en eller flera nya prompts om artikeln innehaller flera tydliga vinklar.
4. Om texten ar tunn/oklar markerar jag vad som behovs for en starkare prompt.

### Multi-prompt-regel

- En artikel kan brytas upp i flera promptfiler om det ger hogre nytta.
- Jag far dela upp i:
  - huvudprompt (breadth)
  - nischprompt (specific use case)
  - remix/variantprompt (annan ton, annat format, annan plattform)
- Alla filer namnsatts alltid som `kategori-promtnamn.md`.

## Prompt-format (standard)

Varje fil innehaller:

- Syfte
- Input
- Instruktioner
- Output-format
- Kvalitetskriterier
- Variant(er)

## Kategorier

Jag valjer kategorier sjalv utifran innehallet (inte bundet till artikelns egna etiketter).

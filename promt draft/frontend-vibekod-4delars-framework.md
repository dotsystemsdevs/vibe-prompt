# frontend-vibekod-4delars-framework

## Syfte
Skapa frontend-prompts som faktiskt levererar genom en tydlig 4-delars struktur.

## Input
- Identity (vad som byggs)
- Audience (vem det ar for)
- Features (funktionalitet)
- Aesthetic constraints (visuell stil + teknik)

## Instruktioner
Kopiera prompten nedan:

```text
Build this UI using the 4-part framework.

Identity:
[what to build]

Audience:
[who this is for]

Features:
- [feature 1]
- [feature 2]
- [feature 3]

Aesthetic + technical constraints:
- Design style:
- Colors:
- Typography:
- Framework/libraries:
- Responsive behavior:

Requirements:
1) Generate production-lean component structure.
2) Include accessibility basics.
3) Keep style consistent with constraints.
4) Add quick verify checklist.

Output format:
[UI Spec]
- ...

[Implementation]
...

[Verify Checklist]
- ...
```

## Output-format
- UI-spec
- Implementation
- Verifieringschecklista

## Kvalitetskriterier
- Hogsignal promptinput
- Mindre generisk UI
- Hoger first-try traeffsakerhet

## Varianter
- Variant A: Strict mode (hard constraints only).
- Variant B: Creative mode (constraints + optional exploration).

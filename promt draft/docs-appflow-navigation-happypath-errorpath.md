# docs-appflow-navigation-happypath-errorpath

## Syfte
Dokumentera appens flow och navigation så AI inte gissar användarresor.

## Input
- Funktioner
- Entry points
- Kritiska användarresor

## Instruktioner
Kopiera prompten nedan:

```text
Create APP_FLOW.md for this product.

Inputs:
- Entry points:
- Core user goals:
- Key screens:

Include:
1) Entry points (primary/secondary)
2) 3-5 core user flows
3) Happy path + error states per flow
4) Decision points and branching logic
5) Validation and error message behavior
6) Mermaid flow diagrams for critical flows

Rules:
- One flow per user goal.
- Document both success and failure paths.
- Keep naming consistent with PRD features.

Output format:
[APP_FLOW]
...
```

## Output-format
- APP_FLOW.md redo att använda

## Kvalitetskriterier
- Kompletta user journeys
- Tydliga branch decisions
- Mindre navigation hallucinationer

## Varianter
- Variant A: Consumer app flows.
- Variant B: B2B dashboard flows.

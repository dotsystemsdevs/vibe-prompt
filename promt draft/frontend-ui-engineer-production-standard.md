# frontend-ui-engineer-production-standard

## Syfte
Generera production-redo frontendlosningar med fokus pa lasbarhet, typning, a11y, prestanda och backend-agnostisk integration.

## Input
- UI-krav
- Framework/stack
- API-kontrakt

## Instruktioner
Kopiera prompten nedan:

```text
Act as a production UI engineer.

Requirements:
[insert requirements]

Standards:
1) Clear component architecture
2) Type-safe interfaces
3) Responsive + accessible design
4) Loading/error/empty states
5) Performance-conscious rendering
6) API-agnostic data abstraction

Output format:
[Architecture]
...

[Component Plan]
...

[Type Interfaces]
...

[Implementation]
...

[Quality Checklist]
- a11y
- performance
- testability
```

## Output-format
- Arkitektur + komponentplan
- Typinterfaces
- Implementation + quality checklist

## Kvalitetskriterier
- Production-standard frontend
- Latt att underhalla
- Smidig backendintegration

## Varianter
- Variant A: React + Tailwind.
- Variant B: Vue + TypeScript.

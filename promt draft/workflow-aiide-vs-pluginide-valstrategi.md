# workflow-aiide-vs-pluginide-valstrategi

## Syfte
Skapa en prompt som avgor nar du ska anvanda en AI-first IDE kontra klassisk IDE med AI-plugin.

## Input
- Teamets nuvarande IDE-stack
- Projektets komplexitet
- Krav pa governance, review och debugbarhet

## Instruktioner
1. Bedom projektets behov av agentiska floden kontra klassisk editor-kontroll.
2. Jamfor AI-IDE och plugin-IDE pa fem axlar: produktivitet, transparens, onboardning, risk, kostnad.
3. Definiera en hybridstrategi for team som vill kombinera bada.
4. Satt tydliga switch-kriterier nar ett spor inte fungerar.
5. Lagg in migrationstips for team som byter verktyg.
6. Ge en kort policytext teamet kan anvanda direkt.

## Output-format
- `Context Snapshot`
- `Decision Axes`
- `Tooling Recommendation`
- `Hybrid Workflow`
- `Switch Triggers`
- `Team Policy Draft`

## Kvalitetskriterier
- Ingen dogmatisk "one tool fits all"
- Tydlig koppling till teamets verkliga arbete
- Praktiska switch-regler
- Enkel att rulla ut i team

## Varianter
- Variant A: Solo developer setup
- Variant B: Multi-dev team med code review-krav

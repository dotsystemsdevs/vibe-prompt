# collaborator-klargor-krav-innan-kod

## Syfte
Tvinga fram klargorande fragor och delad forstaelse innan kodning startar.

## Input
- Problemstatement
- Affarsmal
- Tekniska ramar

## Instruktioner
Kopiera prompten nedan:

```text
Before writing code, ask clarifying questions to remove ambiguity.

Context:
[insert context]

Tasks:
1) Ask up to 7 high-impact clarifying questions.
2) Identify assumptions if unanswered.
3) Propose a draft implementation plan based on current info.
4) Mark what could break if assumptions are wrong.

Output format:
[Clarifying Questions]
1) ...
2) ...

[Current Assumptions]
- ...

[Draft Plan]
1) ...
2) ...

[Assumption Risks]
- ...
```

## Output-format
- Frageset
- Antaganden
- Draft plan + risker

## Kvalitetskriterier
- Mindre rework
- Bättre kravprecision
- Farliga antaganden synliggors

## Varianter
- Variant A: Product feature planning.
- Variant B: Data/ML task scoping.

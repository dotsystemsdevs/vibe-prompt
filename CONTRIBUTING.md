# Contributing to VibePrompt

Thanks for contributing.

## Before You Start

- Read [README.md](./README.md) for project context.
- Read [AGENTS.md](./AGENTS.md) for repo-specific workflow rules.
- Check open issues or open a new one before large changes.

## Local Setup

```bash
npm install
npm run dev
```

Run quality checks before submitting:

```bash
npm run lint
npm run build
```

## Contribution Types

- Prompt content improvements
- Article content updates
- UI/UX fixes
- Bug fixes
- Documentation and structure improvements

## Pull Request Guidelines

1. Keep PRs focused (one logical change per PR).
2. Explain **what changed** and **why**.
3. Include screenshots for UI changes.
4. Link related issue(s) when applicable.
5. Ensure lint and build pass.

## Content Standards

For prompt/article updates:

- Use clear, practical language.
- Prefer examples over abstract advice.
- Add or preserve source references when claims are factual.
- Avoid low-value filler and repeated sections.

## Commit Message Suggestions

Use concise, intent-first messages:

- `docs: improve README open-source structure`
- `content: expand prompting glossary definitions`
- `fix: update broken GitHub links`

## Need Help?

Open an issue with:

- What you are trying to do
- What you expected
- What happened instead
